# Refactoring Plan Generator

Проанализируй кодовую базу и создай структурированный план рефакторинга.

## Шаги

1. Используй Task-агенты для параллельного исследования всех ключевых директорий (src/composables, src/components, src/data)
2. Найди проблемы: дублирование логики, файлы >300 строк, несогласованные паттерны, неиспользуемый код
3. Приоритизируй по импакту: быстрые улучшения первыми, breaking changes последними
4. Выведи нумерованный план с оценкой сложности (S/M/L) для каждой задачи
5. НЕ выполняй никаких изменений — только план
6. Сохрани план в docs/refactoring.md
